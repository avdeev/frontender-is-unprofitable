## 3. Ретроспективный анализ: что НЕ попало в роадмап

Роадмап отражает **устойчивый консенсус** одного автора и сообщества, а не bleeding-edge. Это его достоинство (стабильность) и ограничение (отставание). Ниже — технологии и концепции, которые были важны для индустрии, но либо исчезли из роадмапа, либо никогда в нём не появились.

### 3.1 Технологии, которые были в роадмапе и исчезли

| Технология | Когда была | Когда исчезла | Почему исчезла | Значимость для индустрии |
|---|---|---|---|---|
| **Web Assembly (WASM)** | 2019 (отдельный раздел) | 2024 (перерисовка) | Не уточнено; возможно — нишевый навык для фронтендера | Средняя: WASM не стал мейнстримом во фронтенде, но важен для видео/графики/Figma-подобных приложений. Роадмап 2019 описывал WASM как «бинарные инструкции из высокоуровневых языков... быстрее чем JavaScript». Удаление спорно. |
| **RxJS / ReactiveX** | 2017, 2019 | 2024 (перерисовка) | Reactive-программирование не стало стандартом фронтенда | Низкая для общего роадмапа, высокая для Angular-экосистемы (RxJS — основа Angular). Удаление оправдано для общего роадмапа. |
| **State Management (Redux, MobX, Vuex, ngrx)** | 2017 (Flux, Redux), 2018 (Redux, MobX, ngrx, Vuex, RxJS) | 2024 (нет отдельного раздела) | Hooks (React 16.8, 2019) + React Context + Zustand/Jotai упростили state management; он растворился в фреймворках | Дискуссионно: Redux использовался в 50%+ React-проектов до 2020. Удаление отдельного раздела из роадмапа отражает тренд — state management перестал быть отдельным навыком и стал частью фреймворка. Но для крупных enterprise-приложений Redux Toolkit всё ещё актуален. |
| **Design Patterns / GOF** | 2017 | 2018 (убран) | Считается слишком общим для фронтенд-роадмапа | Низкая: GOF-паттерны не специфичны для фронтенда. |
| **SVG / D3** | 2017 | 2018 (убрано как самостоятельная тема) | Растворилось в фреймворках и библиотеках | Средняя: D3 всё ещё используется в дата-визуализации, но нишевый навык. |
| **Task Runners (Grunt, Gulp)** | 2017 | 2024 (раздел удалён) | npm scripts + bundlers вытеснили task runners | Оправдано: Grunt/Gulp фактически мертвы для новых проектов. |
| **CSS-in-JS (Styled Components, Emotion, Radium, Glamorous)** | 2019 (отдельный подраздел) | 2024 (свернут в примечание) | Tailwind + CSS Modules + vanilla-extract вытеснили CSS-in-JS | Оправдано: State of CSS 2024 показывает CSS Modules (61.1%) > Styled Components (59.1%), но последний всё ещё высок. Сворачивание в примечание — спорно. |
| **Universal / Isomorphic** | 2019 | 2024 | Заменено концепцией SSR/SSG/RSC | Оправдано: «universal» — устаревший термин, заменён на SSR. |
| **After.js** | 2018–2019 | 2023 | Библиотека заброшена | Оправдано. |
| **Flow (type checker)** | 2017, 2019 | 2023 (убран) | TypeScript победил | Оправдано: Flow практически исчез из индустрии. |

### 3.2 Технологии, которые никогда не попали в роадмап

Это самый интересный раздел. Ниже — технологии, которые были важны для фронтенд-индустрии в 2017–2026, но **ни в одной версии роадмапа** не появились.

#### Категория A: Инфраструктура и DevOps (намеренно исключены)

Роадмап фокусируется на фронтенд-навыках, а не на инфраструктуре. Эти технологии важны для фронтенд-разработчика, но сознательно вынесены за рамки:

| Технология | Значимость | State of JS / Stack Overflow | Почему не в роадмапе |
|---|---|---|---|
| **Docker** | Высокая: локальная разработка, CI/CD, deployment | SO 2024: 69% prof devs используют | Считается DevOps-навыком, не фронтенд-специфичным |
| **CI/CD (GitHub Actions, GitLab CI)** | Высокая: каждый современный фронтенд-проект имеет CI | N/A (не в State of JS) | То же — инфраструктурный навык |
| **Kubernetes** | Средняя: редко для фронтендера напрямую | SO 2024: 14% | Энтерпрайз-инфраструктура |
| **Monorepos (Nx, Turborepo, Lerna)** | Высокая и растущая: State of JS 2024 не трекает, но Turborepo → Vercel | Turbopack 10.4% в State of JS 2024 (близкое, но не то же) | Монорепо — архитектурный выбор, не навык; но Nx/Turborepo становятся стандартом для средних и крупных фронтенд-проектов |

> **Вывод:** Роадмап намеренно не включает DevOps. Но в 2026, когда `frontend-roadmap` добавил deployment-платформы (Vercel, Netlify, Cloudflare), граница сдвинулась. Monorepo-инструменты (Nx, Turborepo) — кандидаты на включение в следующую версию.

#### Категория B: Технологии, которые «должны были быть», но не попали

| Технология | Значимость | Данные | Почему не в роадмапе |
|---|---|---|---|
| **Storybook** | Высокая: State of JS 2024 — **39.9%** «used at work», #2 после Jest (62.2%) | [State of JS 2024 testing](https://2024.stateofjs.com/en-US/libraries/testing/) | Storybook — компонент-драйвен разработка, не «тестирование» в классическом смысле. Роадмап не имеет раздела «Component Development / Design Systems Integration». В 2026 добавлена тема `design-systems`, но без Storybook. |
| **Deno** | Средняя: альтернатива Node.js, но не взлетел массово | State of JS write-ins: 8 (2024) | Не стал мейнстримом. Роадмап не включает альтернативы, не получившие adoption. |
| **Bun** (до 2025) | Растущая: runtime + bundler + package manager | State of JS 2024: 35 write-ins; SO 2024: 3.8% | Появился как orphaned content-файл в декабре 2025 (`bun@j2H1qGosbveayviOCVkUs.md`), готовится к включению в новую версию роадмапа. |
| **Micro-frontends** | Средняя: энтерпрайз-архитектура для крупных команд | Не в State of JS | Нишевая архитектура; роадмап фокусируется на индивидуальных навыках, а не на командной архитектуре. |
| **Edge Functions / Edge Computing** | Растущая: Cloudflare Workers, Vercel Edge, Deno Deploy | Cloudflare 15.1% (SO 2024) | Связано с deployment-платформами, которые добавлены в роадмап 2025–2026, но Edge Functions как концепция не выделена. |
| **WebGL / Three.js / Canvas** | Нишевая, но стабильная: 3D-графика в браузере | Не в State of JS (отдельная ниша) | Роадмап не включает 3D-графику — слишком специализированный навык. |
| **Web Workers / Service Workers** | Средняя: Service Workers в роадмапе (PWA-раздел), но Web Workers — нет | — | Service Workers попали через PWA-раздел (2019). Web Workers (general parallelism) не попали, хотя концептуально близки. |
| **RxJS / Reactive Programming** | Высокая для Angular, средняя в целом | Не в State of JS 2024 top testing | Был в роадмапе 2017–2019, убран к 2024. Reactive-программирование не стало универсальным стандартом. |
| **View Transitions API** | Новая (2023+), но перспективная | Chrome 111+ (2023) | Слишком новая для роадмапа; возможно, появится в следующей версии. |
| **Container Queries** | Высокая: меняет адаптивный дизайн | State of CSS 2024 упоминает как часть «New CSS» | Не в роадмапе, хотя CSS-раздел существует. Роадмап фокусируется на инструментах (Tailwind), а не на CSS-фичах. |
| **WebRTC** | Нишевая: видеo/аудио в браузере | Не в опросах | Специализированный API, не массовый навык. |
| **TanStack Query / SWR** | Высокая: server state management — де-факто стандарт для React-приложений | Не в State of JS (отдельная категория) | Server state (React Query/SWR) заменил Redux для многих use cases. Роадмап не имеет раздела «Data Fetching / Server State». Это значительный пробел. |

#### Категория C: Что появился в контент-слое, но ещё не в JSON (42 orphaned-файла)

Эти технологии **уже готовятся к включению** в новую версию роадмапа (см. [frontend-roadmap-evolution.md, раздел «Волны нового контента»](frontend-roadmap-evolution.md#волны-нового-контента-42-осиротевших-файла--без-соответствующих-нод-в-текущем-json)):

- **AI-раздел** (17 тем): AI Agents, Claude Code, Copilot, Cursor, MCP, LLM-провайдеры — крупнейшее расширение с 2019 года
- **Deployment** (6 тем): Vercel, Netlify, Cloudflare, Railway, Render, GitHub Pages — закрытие пробела категории A
- **Новые build tools** (4 темы): Biome, Bun, Rolldown, TanStack Start — обновление тулинга
- **Performance/DevTools** (6 тем): Lighthouse, Cache-Control, Service Workers, Web APIs, DevTools Usage

> **Bun** и **Deployment-платформы** — из категории B (отсутствовали) переходят в роадмап. **Deno** — остаётся за рамками (не набрал adoption).

### 3.3 Почему роадмап «отстаёт» от индустрии

Анализ показывает **3 причины отставания**:

#### Причина 1: Консенсус vs хайп

Роадмап отражает **устойчивый консенсус**, а не хайп. Технология должна доказать viability, прежде чем попадёт в роадмап.

- **Пример:** Vite появился в роадмапе в 2023, хотя выпущен в 2020 и набрал adoption к 2022. Задержка ~2 года.
- **Пример:** Bun появился в контенте в декабре 2025, хотя выпущен в 2023 и набрал adoption к 2024. Задержка ~2 года.
- **Пример:** AI-ассистенты (Copilot) появились в роадмапе в 2026, хотя Copilot preview — 2021, ChatGPT — 2022, массовое adoption — 2023–2024. Задержка ~3 года.

> **Вывод:** Роадмап отстаёт на **2–3 года** от реального adoption. Это осознанный выбор — фильтрация хайпа.

#### Причина 2: Индивидуальный vs командный навык

Роадмап описывает **индивидуальный** путь обучения, а не командную архитектуру. Поэтому исключены:
- Micro-frontends (командная архитектура)
- Monorepos (командная структура)
- CI/CD (инфраструктурный навык)
- Docker (инфраструктурный навык)

> **Вывод:** Роадмап фокусируется на «что должен знать один разработчик», а не на «как работает команда». Это ограничивает полноту, но сохраняет фокус.

#### Причина 3: Разделение структуры и контента (2024+)

С 2024-06-23 визуальный JSON (`frontend.json`) **не менялся** — 137 нод, те же ID. Эволюция продолжается через **слой контента** (см. [frontend-roadmap-evolution.md, «Архитектура управления контентом»](frontend-roadmap-evolution.md#архитектура-управления-контентом)):

- Внешняя БД roadmap.sh → GitHub Actions → 120 content-файлов
- 42 «осиротевших» файла готовят новую версию, но JSON ещё не обновлён
- Очистка (2026-05-14) удалила 23 старых файла

> **Вывод:** Роадмап находится в **переходном состоянии** (июль 2026). Новая версия визуальной карты готовится, но не опубликована. Отставание в этот период — артефакт процесса синхронизации, а не сознательное решение.
